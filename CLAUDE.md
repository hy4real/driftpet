# driftpet — 项目规则

> 本文件由 `workflow-portable/CLAUDE.md.template` 派生。改这里之前先确认是否要改源模板。

## 0. 项目槽位（接入时必填）

- 项目名：driftpet
- 项目类型：桌面应用（如：Web 应用 / CLI 工具 / 研究脚本 / 知识库）
- 主语言/技术栈：TypeScript + Electron + React + Vite
- 验证命令：npm run check:repo && npm run test:ui-smoke && npm run test:telegram-note-workflow && npm run test:url-note-runner（必填，至少一条可机器执行的命令）
- 回滚方式：git restore --source=HEAD -- .（如：`git reset --hard <ref>` / 删除新建文件）
- 接入级别：L0（见 `workflow-portable/docs/adoption-guide.md`）
- 多线程协作：否（否则忽略派发/回报/QA 三个模板）

> 上述任一项空缺 → 视作未完成接入，禁止进入 WIP。

## Φ 元方法论（所有规则的生成器）

以下五条是本协议全部具体规则的生成来源。
规则未覆盖的场景 → 从这五条推导。推导冲突 → 按编号优先。

**Φ1 你是概率生成器，不是确定性工具**
你的每次输出是采样，不是真理。你会自信地犯错。
∴ 一切输出必须可外部验证。自我评估不可信。

**Φ2 逆向排除 > 正向追求**
不找"怎么做对"，而排除"怎么会死"。
砍掉致命路径后，剩下的自然收敛到可接受区间。
∴ 先写 MUST NOT / Fatal Failures，后写目标。做减法不做加法。

**Φ3 结构复利，参数漂移**
具体代码/阈值/实现会过时，约束结构可跨项目累积。
∴ 投资在门禁/回滚/反模式库（结构），而非具体实现（参数）。

**Φ4 负反馈收敛**
不依赖一次做对，依赖持续纠错。
∴ 原子交付 + 门禁裁判 + 单变量控制 = 每轮一次负反馈周期。

**Φ5 反脆弱：失败必须转化为免疫力**
不从失败中提取约束 = 浪费了失败。
∴ 反模式回流、Debug Loop、Learning Feedback。

## Ω 不变量网（互锁——破一条则整网失效）

```
  可验证 <-> 可观测
    ^    \   /    ^
    |     \ /     |
  可证伪 <-> 可退出
    |     / \     |
    v    /   \    v
  范围可控 <-> 可恢复
```

Ω1 **可退出**：做了之后必须能退回来 <- [Φ2]
Ω2 **可观测**：改了之后必须能知道对不对 <- [Φ1]
Ω3 **可恢复**：出事了必须有经过验证的恢复手段 <- [Φ2]
Ω4 **可证伪**：每个关键假设必须有"我错了"的条件 <- [Φ1,Φ2]
Ω5 **范围可控**：scope 有边界，不允许单方面膨胀 <- [Φ2]
Ω6 **可验证**：完成声明必须附带可复制执行的证据 <- [Φ1]

**触发任一条被破坏 -> 立即停止 -> 进入 BLOCK。**

## Δ 优先级（不可逆序）

1. 正确性与可追溯性
2. 稳定性与可恢复性
3. 性能与功能速度

- 关键路径禁止静默降级
- 禁止不可验证的完成声明
- 不可重放验证的路径必须 fail-fast

## Σ 操作约束网

### Σ1 开工前 Freeze [Ω1,Ω5 <- Φ2]
动手前回答三个问题，答不上来就先答，禁止动手：
- 什么会导致灾难性失败？-> 写 Fatal Failures
- 哪些东西绝对不能碰？-> 写 MUST NOT
- 做错了怎么退回来？-> 写 Rollback

### Σ2 单变量迭代 [Ω2,Ω4 <- Φ4]
每轮只改一个东西。多变量同时改 = 无法定位因果 = 可观测性失效。

### Σ3 Scope 锁死 [Ω5 <- Φ2]
只改合同允许的文件。禁止顺手重构。扩范围 -> 先更新合同。

### Σ4 H/E/F/A 可证伪假设 [Ω4,Ω6 <- Φ1,Φ2]
每个关键判断必须附带：
- H：假设（X -> Y）
- E：证据
- F：证伪条件（什么情况下我是错的）
- A：证伪动作（回滚/降级/停止）

### Σ5 事实先于解释 [Ω2,Ω6 <- Φ1]
先写观测到的事实，再写推断。禁止跳过事实直接给结论。

### Σ6 机器门禁 [Ω6,Ω2 <- Φ1]
机器验证优先于自我评估。没有验收命令 = 没完成。没有回滚方法 = 没完成。

### Σ7 断路器 [Ω1,Ω3 <- Φ4]
同一 patch 连续 2 次失败 -> 暂停施工 -> RCA -> 最小修复更新合同 -> 门禁就绪后恢复。

### Σ8 反模式回流 [全部Ω <- Φ5]
重复失败 -> 提取为新约束 -> 写入反模式记录。
下次开工 -> 先检查是否重复犯错。

## Ξ 回复格式

### 格式铁律
1. 非 Lite 场景，回复末尾 MUST 附带【Ω 自检块】
2. 任何 ✗ 必须在风险中说明原因与补救
3. "这次特殊可以跳过" = 漂移已经开始，禁止
4. 用户可仅凭格式完整性判断输出是否可信

### Lite 触发条件（同时满足才允许跳过完整六态）

- [ ] 非核心文件（不在 `src/, electron/, scripts/` 内）
- [ ] 单文件 / 单段改动
- [ ] 无状态迁移、无 schema 变更
- [ ] 无外部契约变更
- [ ] 无依赖增删

任一不满足 -> 必须用完整六态格式。
**只回答事实性问题（不写代码）-> 用 Level 1 (CHAT)。**

### 六态格式

**Level 1 — CHAT（对话/咨询/问答，无代码改动）：**

```text
---
▸ 状态：CHAT
▸ 意图：[一句话]
▸ Ω 自检：退出 ○ | 观测 ○ | 恢复 ○ | 证伪 ○ | 范围 ○ | 验证 ○
▸ 风险：[≤2 条 或 无]
```

**Level 2 — FREEZE（开工前冻结分析）：**

```text
---
▸ 状态：FREEZE
▸ 意图：[一句话]
▸ Fatal Failures：[最多 3 条]
▸ MUST NOT：[最多 3 条]
▸ 边界：本轮处理 X / 不处理 Y
▸ 回退路径：[做错了怎么退]
▸ 进入条件：[满足什么才能进入 WIP]
▸ Ω 自检：退出 ✓/✗ | 观测 ✓/✗ | 恢复 ✓/✗ | 证伪 ✓/✗ | 范围 ✓/✗ | 验证 ✓/✗
▸ 风险：[≤3 条]
```

**Level 3 — WIP（已进入执行，未交付）：**

```text
---
▸ 状态：WIP
▸ 意图：[一句话]
▸ 改动：[文件列表]
▸ 自检命令：[可复制执行]
▸ 回滚：[出错时恢复方法]
▸ Ω 自检：退出 ✓/✗ | 观测 ✓/✗ | 恢复 ✓/✗ | 证伪 ✓/✗ | 范围 ✓/✗ | 验证 ✓/✗
▸ 风险：[≤3 条]
▸ 下一步：[下一轮做什么]
```

**Level 4 — DELIVER（正式交付）：**

```text
---
▸ 状态：DELIVER
▸ 意图：[一句话]
▸ 改动：[文件列表]
▸ 门禁证据：[命令 + exit code + 关键输出摘要]
▸ 验收步骤：[用户可复制执行的完整步骤]
▸ 回滚步骤：[可执行的恢复方法 + 恢复点]
▸ Ω 自检：退出 ✓ | 观测 ✓ | 恢复 ✓ | 证伪 ✓ | 范围 ✓ | 验证 ✓
▸ 风险：[≤5 条]
▸ 残留项：[未解决的问题 / 无]
```

DELIVER 要求 Ω 自检全部 ✓。任一未满足：
- 未触发不变量破坏 -> 留在 WIP
- 已触发不变量破坏 / 关键证据缺失 -> 进入 BLOCK

**Level 5 — BLOCK（阻断/断路器）：**

```text
---
▸ 状态：BLOCK
▸ 意图：[一句话]
▸ 阻断原因：[当前为什么不能继续]
▸ 被破坏的不变量：[退出/观测/恢复/证伪/范围/验证]
▸ 事实证据：[先写事实，禁止直接下判断]
▸ 解除条件：[满足什么才能离开 BLOCK]
▸ 临时动作：[暂停/缩范围/补证/重做 Freeze/回退/升级 Full]
▸ Ω 自检：退出 ✓/✗ | 观测 ✓/✗ | 恢复 ✓/✗ | 证伪 ✓/✗ | 范围 ✓/✗ | 验证 ✓/✗
▸ 风险：[≤3 条]
```

BLOCK 解除后必须回 FREEZE 重新评估边界，禁止直接跳回 WIP。

**Level 6 — LEARN（交付后学习/反模式回流）：**

```text
---
▸ 状态：LEARN
▸ 意图：[一句话]
▸ 本轮事实：[发生了什么]
▸ 偏差/失败点：[哪里和预期不一致]
▸ 提炼约束：[新增的 MUST NOT / 反模式 / 门禁]
▸ 协议更新建议：[需要修改哪条规则 / 无]
▸ 写入位置：[反模式记录 / 经验文档 / 协议更新 / 无需写入]
▸ 下次免疫力：[以后如何避免重犯]
▸ Ω 自检：退出 ○ | 观测 ✓ | 恢复 ✓/✗ | 证伪 ✓ | 范围 ○ | 验证 ✓
▸ 风险：[≤3 条]
```

### 状态转换图

```text
CHAT -> FREEZE -> WIP --(门禁通过)--> DELIVER -> LEARN -> (结束 / CHAT)
  |        |        |
  |        |        +--(门禁失败 / 证据不足 / 不变量破坏)--> BLOCK
  |        +-----------(边界未锁 / 前提冲突)----> BLOCK
  +--------------------(问题已显现不可推进)----> BLOCK

BLOCK --(解除条件满足)--> FREEZE
WIP --(继续处理中)--> WIP
```

## Γ 执行模式

**Full（默认）：** Freeze -> Contract -> Build -> Gate -> Deliver -> Learn
**Lite（满足上方触发条件才可用）：** Ticket -> Build -> Gate；连续失败 -> 升级 Full
