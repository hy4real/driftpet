# 任务派发模板

总控向真实线程派发任务时使用。整段必须用**可直接复制的 Markdown 代码块**输出。

适用：L1 / L2 / L3 接入级别。

---

## 标准派发模板

```md
执行线程：
- <线程名称>

任务ID：
- <MMDDTxxx>

标题：
- <任务标题>

简短描述：
- <1-2 句摘要>

状态：
- 待执行

执行顺序：
- 当前唯一任务 / 可并行

前置任务：
- 无 / <前置任务ID>

scope：
- <允许修改的文件、模块、目录>

out_of_scope：
- <禁止顺手修改的文件、模块、目录>

acceptance：
- <通过标准 1（必须可机器验证）>
- <通过标准 2>

risk_boundary：
- <风险边界 1：什么禁止做>
- <风险边界 2：什么情况必须停下进入 BLOCKED>

verify：
- <本轮跑什么命令证明完成>

rollback：
- <出错时如何恢复，必须可执行>

按线程规则执行：
- <规则文档路径或名称>

是否需要提交代码：
- 需要 / 不需要

是否进行QA验收：
- 是 / 否（L2+ 才填，L1 默认 否）

QA验收方式：
- 正常验收 / 免内容验收 / 无

规则更新提醒：
- 无 / 请先阅读规则文档后再执行本任务。
```

---

## 字段说明

### 必填字段（所有级别）

- `执行线程` `任务ID` `标题` `状态`
- `scope` `out_of_scope`：Σ3 Scope 锁死的体现，必须显式
- `acceptance`：Ω6 可验证的体现，至少一条可机器验证
- `risk_boundary`：Σ1 Freeze 的"什么不能碰"
- `verify` `rollback`：Ω2/Ω3 的硬要求

### L2+ 补充字段

- `是否进行QA验收`：是 / 否
- `QA验收方式`：见下方"QA 模式"

### L3 补充

L3 接入级别下，派发文本应同步生成 `task-packet.json` 副本，存入项目运行时目录（推荐路径：`.workflow/tasks/<task-id>.json`）。

---

## 派发铁律

1. **默认一次只正式派 1 个任务**
2. 多个任务彼此独立、可并行才允许批量派 → 必须显式写 `执行顺序：可并行`
3. 有依赖关系 → 不要批量派
4. 任务边界必须明确（线程 / 文件范围 / 验证方式 / 是否提交代码 / 是否进 QA）
5. 不允许静默绕开流程

---

## QA 模式（L2+）

### 正常验收

```md
是否进行QA验收：
- 是

QA验收方式：
- 正常验收
```

### 免内容验收（流程节点已走完，但内容由总控判断）

```md
是否进行QA验收：
- 是

QA验收方式：
- 免内容验收
```

QA 仍然要更新 QA 文档，但只确认流程节点完成，不对任务内容做最终判断。

### 当前暂不进 QA

```md
是否进行QA验收：
- 否

QA验收方式：
- 无
```

---

## 任务 ID 规则

- 格式：`MMDDTxxx`，例如 `0510T001`
- 一次正式派发对应一个 ID
- 不同轮次不能复用同一 ID
- 派发、执行、测试、QA 全程使用同一 ID

---

## 规则更新提醒

工作流规则刚更新时，每个活跃线程的下一条任务必须带：

```md
规则更新提醒：
- 请先阅读规则文档后再执行本任务。
```

同一轮里对同一线程只提醒一次。
